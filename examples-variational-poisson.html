Introduction

To introduce the Rodin::Variational module it is important to start from a type of "Hello World!" example. In this context, the Poisson equation fits exactly our purpose. Hence, in this example we will seek to solve:

In general, when solving an equation like this we usually utilize its variational formulation. Multiplying by a test function and integrating by parts we may write the variational formulation

Obtaining the variational (or weak) formulation of a partial differential equation is a very standard procedure in the Finite Element Method literature. Hence we won't go into specifics. Rather, we deem it best to show how to implement said weak formulation and solve it.

Includes and Namespaces

In order to start using Rodin we first need to include the relevant parts which will aid us in the resolution of our equation. If you are familiar with finite elements then you will know that in particular we will need three main components:

  • A mesh approximating our domain .
  • A solver object that allows us to solve the problem
  • A way to specify the problem

Hence we import the required modules in Rodin:

#include "Rodin/Mesh.h"
#include "Rodin/Solver.h"
#include "Rodin/Variational.h"

using namespace Rodin;
using namespace Rodin::Variational;

The first part includes the following modules:

  • Rodin::Mesh
  • Rodin::Solver
  • Rodin::Variational

While the second part will simply introduce all the objects inside the Rodin and Rodin::Variational namespaces to the global scope, so we don't have to write their namespace again.

Implementing the solution

Next we would like to write the code which actually solves the equation. For our purposes we will write it inside the main function like so:

int main()
{
  // Code solving the equation
}

One of the first things you usually do is to keep track of the segment of the boundary where we impose the boundary conditions. In this case, we have homogenous Dirichlet boundary conditions on the boundary elements with attribute equal to 1. Additionally, we load the mesh from some file given its filename:

int Gamma = 1;
Mesh Omega = Mesh::load(filename);

With the mesh loaded, we can build our finite element space and create some functions belonging to it. For our purposes, we need that our space contains functions belonging to the collection of functions. This is achieved by the following line of code:

H1 Vh(Omega);

After creating this space, we construct a trial function which will represent our solution:

GridFunction u(Vh);

Having built our finite element space and declared our trial function, we can start implementing the weak formulation associated to the Poisson equation. In general, Rodin expects that you write you specify the weak formulation by moving all terms to the left hand side of the equation. In symbols, this would look like:

The left hand side is then used to define the variational problem which we want to solve. In the code this is achieved by the following declaration:

auto f = ScalarCoefficient(1.0);
auto g = ScalarCoefficient(0.0);

Problem poisson(u);
poisson = DiffusionIntegrator()
        - DomainLFIntegrator(f)
        + DirichletBC(Gamma, g);

In this case the DiffusionIntegrator() object represents the bilinear term

while the DomainLFIntegrator(f) object represents the linear term

Lastly, the DirichletBC(Gamma, 0) object specifies that the solution will satisfy the homogenous Dirichlet condition:

Generally, it is somewhat tedious to remember what each integrator represents so the developer is encouraged to use search the documentation with the integrator term to know what the integrator represents.

With the problem defined, we now need to solve the underlying linear system given by the associated stiffness matrix. There are many different types of solvers in Rodin but for this example we will utilize the Preconditioned Conjugate Gradient (PCG) solver:

Solver::PCG().setMaxIterations(200)
             .setRelativeTolerance(1e-12)
             .printIterations(true)
             .solve(poisson);

This piece of code creates a PCG object and sets some parameters before solving the problem we just defined.

Finally, we will typically want to visualize our solution. We can do this by saving both the solution and mesh objects to file, and then using GLVis to view the results.

u.save("u.gf");
Omega.save("Omega.mesh");

To visualize the results, we can call GLVis from the command line:

glvis -m Omega.mesh -g u.gf
